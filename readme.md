#Задание  
Разработать пошаговую форму регистрации на angularjs, разрешается использование сторонних библиотек.  
Каждый следующий шаг становится доступным при заполнении всех необходимых полей.  

###Примечание:  
1) Страница содержит меню с перечислением шагов;  
2) Активный шаг подсвечивается;  
3) Доступный шаг подсвечивается;  
4) На каждой странице есть кнопка далее, активная после заполнения всех необходимых полей;  
5) Для полей Email, День рождения, Телефон необходима валидация;  
6) Навигация, роутинг;  
7) Вывод сообщений об ошибках.  

##Шаг 1 Авторизация:
- Логин (обязательно);  
- Email (обязательно) (валидация);   
- Пароль (обязательно);  
- Подтверждение пароля (обязательно);  

##Шаг 2 Персональные данные:
- Имя (обязательно);
- Фамилия (обязательно);
- День рождения:
     - День (обязательно); 
     - Месяц (обязательно); 
     - Год (обязательно); 
- Пол (обязательно);
- Дополнительная информация;  

###Примечание:
1) Для выбора даты можно использовать сторонний виджет.  
2) Дополнительная информация максимум 512 символов, необходимо отображать количество оставшихся символов.  

##Шаг 3 Контактная информация:  
- Телефон (обязательно);
- Страна (обязательно);
- Город (обязательно);
- Адрес (обязательно);
- Адрес;
- Социальные сети:
     - vkontakte
     - facebook
     - twitter
     - instagram
     - google+  
     
###Примечание:  
1) Формат ввода телефона "(###) ###-##-##"  
2) Для заполнения доступно два поля "адрес". Если первое не заполнено, второе не доступно.  

##Шаг 4 Отправка данных:
- Подтверждение правил.  

###Примечание:  
1) На странице делается вывод всей введенной информации.  

##Дополнительными плюсами будет:
1) Использование Angular Animation где это может быть необходимо;  
2) Использование Directive где это необходимо;  
3) Сделать свой дизайн для интерфейса, максимальное юзабилити;  
4) Валидация URL, т.е. не разрешать переход на следующий шаг в форме изменив URL, если предыдущие шаги не были валидно заполнены;  
5) Сохранение данных в localstorage.
